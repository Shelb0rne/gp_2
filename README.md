## Бизнес-задача

Когда клиент приезжает в автосалон сдать машину по trade-in, оценщик тратит 15-30 минут на ручную оценку. Это долго, субъективно и приводит к ошибкам — либо занижают цену (клиент уходит), либо завышают (дилер теряет деньги).

**Цель проекта** — собрать датасет с объявлениями о продаже б/у автомобилей и данными о безопасности, чтобы потом на его основе построить модель, которая за секунды предсказывает рыночную цену авто по его характеристикам и описанию продавца.

**Целевая переменная:** price (цена в рублях, числовая, регрессия)

**Текстовый столбец:** description (описание от продавца на Drom.ru)

**Потенциальный заказчик:** автодилер, trade-in сервис (CarPrice, Fresh Auto)


## Источники данных
### 1. Drom.ru, Bibika.ru — Web Scraping (requests + BeautifulSoup, Selenium)

Drom.ru — один из крупнейших автомобильных классифайдов в России (500 000+ объявлений). Мы собрали данные с объявлений о продаже б/у автомобилей

Bibika.ru - поисковая система по продаже и покупке новых и подержанных автомобилей

**Что собрали:**
- Заголовок, цена, год выпуска
- Марка, модель, город
- Двигатель, мощность (л.с.), КПП, привод, кузов, цвет, руль
- Пробег (км)
- Описание от продавца (текстовый столбец)

**Итого:** 10 506 объявлений, 36 марок, 16 столбцов

**Файл:** `url_final.csv`

### 2. NHTSA API — данные о безопасности (API)

NHTSA (National Highway Traffic Safety Administration) — Национальная администрация безопасности дорожного движения США, агентство Министерства транспорта США.
В проекте используется бесплатный открытый API NHTSA без API-ключа, который позволяет получать данные об отзывных кампаниях, жалобах потребителей, рейтингах краш-тестов и производителях автомобилей.

**Используемые эндпоинты (6 штук):**

|   | Эндпоинт | Что возвращает |
|---|----------|---------------|
| 1 | `/recalls/recallsByVehicle` | Отзывные кампании по марке/модели/году |
| 2 | `/complaints/complaintsByVehicle` | Жалобы потребителей по марке, модели и году |
| 3 | `/SafetyRatings/modelyear/.../make/.../model/...` | Поиск рейтинга безопасности |
| 4 | `/SafetyRatings/VehicleId/...` | Детальные оценки краш-тестов |
| 5 | `/vehicles/GetModelsForMakeYear/...` | Список моделей марки за год |
| 6 | `/vehicles/GetManufacturerDetails/...` | Информация о производителе |

**Что получили:**
- `recalls_count` — количество отзывных кампаний
- `complaints_count` — количество жалоб потребителей
- `injuries`, `deaths` — число травм и смертей в жалобах
- `has_fire`, `has_crash` — были ли пожары/аварии
- `overall_rating`, `front_crash_rating`, `side_crash_rating`, `rollover_rating` — рейтинги безопасности (1-5 звёзд)
- `manufacturer_country` — страна производителя
- `model_exists` — удалось ли нйти модель в базе данных
- `matched_model_name` — название модели, с которой удалось сопоставить автомобиль
- `model_match_status` — статус сопоставления модели

**Особенность реализации:**

Перед запросами к отзывам, жалобам и рейтингам в коде выполняется отдельная проверка существования модели через GetModelsForMakeYear. Для этого используется нормализация названия модели и мягкое сопоставление, чтобы уменьшить потери данных из-за различий в написании модели.

### 3. MarketCheck API — рыночные данные по автомобилям

MarketCheck — коммерческий API для анализа автомобильного рынка.
В проекте он используется для обогащения датасета рыночными признаками по year, make, model. В отличие от NHTSA, этот API требует API-ключ.

**Используемый эндпоинт (1 штука):**

|  | Эндпоинт | Что возвращает |
|---|----------|---------------|
| 1 | `/v2/search/car/active` | Агрегированную статистику по активным объявлениям автомобилей|

**Что получили:**
- `marketcheck_avg_price` — средняя цена найденных активных объявлений
- `marketcheck_avg_days_on_market` — среднее время нахождения автомобиля в продаже
- `marketcheck_popularity_score` — количество найденных активных объявлений, используемое как индикатор популярности модели на рынке

**Итого:** 10 013 записей, 21 столбец

**Файл:** `final_data.csv`

## Инструменты

### Основные (с семинара)
- `requests` — HTTP-запросы к сайтам и API
- `BeautifulSoup` — парсинг HTML-страниц
- `pandas` — работа с таблицами
- `re` — регулярные выражения для извлечения данных
- `tqdm` — прогресс-бар для циклов
- `time.sleep` — задержки между запросами

### Дополнительные (не с семинара)
- `Selenium` — для сбора динамического контента (фильтры поиска, кнопка "Показать ещё" в отзывах, подгрузка фото при скролле)
- `oxylabs` — прокси для автоматической смены ip адреса 
- `dotenv` — утилита для работы с дополнительными переменными среды
- `difflib` — модуль для близкого сравненния марки и модели автомобилей
- `logging` — создание логгера

 
### Логирование
Логирование настроено в Python через модуль `logging` в файле `API/logs/logger.py`.

Используется один файловый обработчик:

- `logs.log` — в него записываются сообщения уровня INFO, WARNING и ERROR

**Формат записи:**
- время
- имя логгера
- уровень сообщения
- текст сообщения

Логи сохраняются в файл: `API/logs/logs.log`


## Как запустить

### 1. Установка зависимостей
```bash
pip install requests beautifulsoup4 lxml pandas tqdm fake-useragent pyyaml selenium curl_cffi cloudscraper dotenv difflib 
```

### 2. Скрейпинг Drom.ru
Открыть `Scraping/Парсинг.ipynb` и выполнить ячейки по порядку. Скрипт:
- Собирает ссылки на объявления по маркам (15 страниц на марку)
- Парсит каждое объявление функцией `GetCar()`
- Сохраняет чекпоинты каждые 50 записей
- При бане (429) автоматически останавливается и сохраняет прогресс

Заполнить файл `.env` переменные:
- `USERNAME`
- `PASSWORD`

[Получить данные](https://dashboard.oxylabs.io/en/overview/web-unblocker)

Открыть файл `Scraping/Сбор_машин_автосмена_IP.ipynb`
Выполнить все ячейки по порядку. Скрипт:
- Для указанного набора ссылок собирает информацию о машине из объявления. 
- Используеn requests для чтения страниц
- oxylabs для автоматической смены ip 

В итоге получаем датасет с информацией об авто из объявлений `url_final`

### 3. Selenium
Открыть `Scraping/Selenium_parsing.ipynb`. Используется для чтения информации с сайта https://www.bibika.ru/catalog/
- Собираются ссылки на объявления по указанной марке 
- Формируется итоговый список объявлений из нескольких марок 
- Запускается сбор данных об авто из каждого объявления 

В итоге получается датасет с информацией об авто по каждой ссылке `selenium_cars_final.csv`

### 4. API
Заполнить `.env` пременную:

-`API_KEY`

[Получить ключ](https://universe.marketcheck.com/)

Открыть и запустить `API/api.py`. Для каждой сроки берется комбинация марка/модель/год из датасетов `url_final.csv` и `selenium_cars_final.csv`, выполняется 6 запросов к NHTSA API и один запрос к MarketCheck API. В скрипте настроены автопоторы запросов в случае возникновении ошибки, а так же активно ведется логирование процесса выполнения. После прохода по всем строкам датасетов полученные данные сохраняются в csv файлы `final_data.csv` и `selenium_final_data.csv`

Логи выполнения скрипта доступны в `API/logs/logs.log`

### 5. Univariate Analysis
Открыть `EDA/ГП_2_EDA_9.ipynb`. Выполнить все ячейки последовательно. Ноутбук объединяет данные в один датасет для анализа признаков. Так же происходит очистка пропусков и корректировка кривых значений. По итогу выполнения ячеек формируются три датасета:
- `df_main.csv` - главный датасет с основными признаками автомобилей
- `df_marketcheck.csv` - датасет с данными из MarketCheck API
- `df_rating.csv` - датасет с данными из NHTSA API

### 6. Multivariate Analysis
Открыть `EDA_MULTI/ГП_2_Мультивариант.ipynb`. Выполнить все ячейки последовательно. Ноутбук проводит анализ признаков из трех датасетов предыдущего этапа.

**Ключевые выводы:**
- Черный, белый и серый цвета самые ликвидные цвета

## Сложности и как их решали

### Бан Drom.ru (429 Too Many Requests)
Drom.ru активно банит ботов по IP. Мы столкнулись с блокировками при массовом сборе данных.

**Что пробовали:**
- Увеличение задержек между запросами (sleep 3-7 сек)
- Ротация User-Agent
- Порционный сбор (по 50 записей с перерывами)
- Смена IP через Oxylabs прокси
- Автоматические чекпоинты каждые 50 записей

**Как решали:**
- Смена IP через Oxylabs прокси
- Автоматические чекпоинты каждые 50 записей

### Ошибки NHTSA

Во время работы с NHTSA API мы столкнулись с частыми временными ошибками со стороны сервиса: 503 Service Unavailable, SSLError, а также с обрывами соединения при HTTPS-запросах. Из-за этого часть запросов могла завершаться неуспешно даже при корректных параметрах.

**Как решали:**

- Функция безопасного запроса с автоматическими повторами. Если запрос завершается с временной ошибкой, программа делает небольшую паузу и повторяет попытку несколько раз.
